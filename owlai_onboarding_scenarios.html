시나리오 1 · 로컬 파일

로컬 파일 직접 업로드

김M

김민지 — 인사팀 주임 (비개발자)

취업규칙·복리후생 PDF 등 사내 규정 문서를 자주 찾아보지만 파일이 여러 폴더에 흩어져 있음. AI 경험 거의 없음. "그냥 물어보면 답이 나왔으면 좋겠다"

비개발자 단독 사용자 즉시 사용 기대 파일 수 5개 이하
1

가입 직후 — 빈 채팅 화면 첫 진입

로그인하면 채팅 창이 바로 열린다. "파일을 올리거나, 바로 질문해도 돼요"라는 안내 문구 하나만 표시. 별도 설정 화면 없음.

설계 포인트 온보딩 마법사·스텝 안내 없이 채팅 창이 기본 화면. 설정을 강제하지 않는다.
2

파일 드래그앤드롭 또는 클립 아이콘 클릭

채팅 입력창에 파일을 끌어다 놓거나 클립 아이콘 클릭. PDF, Word, Excel, HWP 지원 표시. 업로드 후 "이 파일 기반으로 대화할게요"라고 채팅에서 자동 응답.

성공 신호 파일명과 페이지 수가 채팅 말풍선에 표시되면 사용자가 "읽혔다"고 인식한다.
3

AI가 파일 요약 + 예시 질문 3개 제안

"취업규칙 문서네요. 이런 것도 물어볼 수 있어요 →" 형태로 샘플 질문 버튼을 제공. 사용자는 버튼만 눌러도 첫 대화를 시작할 수 있다.

설계 포인트 첫 질문을 사용자가 직접 타이핑하지 않아도 되게 함. 진입 장벽 최소화.
4

파일 저장 여부 선택 (대화 종료 시점)

"이 파일을 나중에도 쓸 수 있게 저장할까요?" 토스트 알림. 저장하면 '내 자료' 탭에 보관되어 다음 대화에서 바로 선택 가능.

주의 저장을 기본값으로 강제하지 말 것. 민감 파일이 모르게 저장되는 것에 대한 불안감이 이탈 원인이 된다.
5

실패 케이스 — 암호화·손상 파일

에러 코드 대신 자연어 안내: "이 파일은 암호가 걸려 있어 내용을 읽기 어려워요. 암호를 해제한 뒤 다시 올려주세요." 스캔 이미지라면 "텍스트 추출을 시도할게요 (약 30초)"로 대체 경로 제공.

설계 포인트 에러가 생겨도 다음 행동을 명확히 안내. 막힌 느낌을 주지 않는 게 핵심.
NAS / 공유 드라이브 연동
시나리오 2 · NAS 연동

사내 NAS / 공유 드라이브 연동

박T

박성호 — 기술지원팀 팀장 (준개발자)

NAS에 매뉴얼·이슈 로그 수백 개 보관. 팀원들이 같은 질문을 반복. "폴더째로 연결해서 팀 전체가 쓸 수 있으면 좋겠다"

팀 관리자 다수 파일 정기 업데이트 필요 IT 협조 필요할 수 있음
1

연동 시작 — '데이터 소스 추가' 진입

'내 자료 > 데이터 소스 추가' 클릭. 소스 선택 화면에서 "어디에 파일이 있나요?" 질문 형태로 제시: 로컬 파일 / NAS·네트워크 드라이브 / 그룹웨어 / 클라우드 드라이브.

설계 포인트 기술 용어 대신 질문 형태. NAS를 모르는 사용자도 "네트워크 드라이브"는 인식 가능하다.
2

연결 방식 선택 — 에이전트 설치 vs 주소 직접 입력

옵션 A: OwlAI 에이전트 프로그램 PC 설치 (NAS 접근 위임). 옵션 B: SMB/NFS 주소 + 계정 직접 입력. 일반 사용자는 A에 '권장' 배지 표시.

IT 협조 필요 시 "IT 담당자에게 요청 보내기" 버튼 → 담당자에게 메일·슬랙 알림 자동 발송. 사용자는 대기 상태로 전환되어 직접 설정할 필요 없음.
3

폴더 선택 — 탐색기 형태 UI

NAS 연결 완료 후 폴더 트리 표시. 체크박스로 동기화할 폴더 선택. 하위 폴더 포함 여부 토글 제공. 선택 즉시 예상 파일 수·용량 미리 표시 ("약 340개 파일, 2.1 GB")

차별화 포인트 웍스AI는 하위 폴더 동기화 불가. OwlAI는 재귀 동기화 지원을 화면에서 명시적으로 보여줌.
4

최초 인덱싱 — 진행 상황 실시간 표시

"매뉴얼_v3.pdf 처리 중 (128/340)..." 형태로 파일별 진행 상황 표시. 완료 예상 시간 안내. 인덱싱 중에도 완료된 파일은 바로 질문 가능.

설계 포인트 전체 완료를 기다리지 않아도 됨. 일부라도 쓸 수 있으면 대기 중 이탈 방지.
5

동기화 주기 설정 + 팀 공유

동기화 주기 선택: 실시간 / 매일 / 매주. 팀 공유 범위 설정: 나만 / 팀 전체 / 특정 멤버. 공유 시 팀원에게 "새 데이터 소스가 추가됐어요" 알림 발송.

권한 설계 파일 접근 권한은 원본 NAS 권한 상속. 접근 불가 파일은 인덱싱에서 자동 제외 + "X개 파일은 권한 없어 제외됐어요" 안내.
6

실패 케이스 — 연결 끊김 / 동기화 실패

NAS 연결이 끊기면 관리자에게 알림: "기술지원_매뉴얼 폴더와의 연결이 끊겼어요. 마지막 동기화: 어제 오전 9시." 재연결 버튼 + 원인 안내 제공.

vs 웍스AI 웍스AI는 동기화 실패 시 "OpenAI 문제라 해결 불가"라고 명시. OwlAI는 원인을 특정하고 재시도 경로를 제공한다.
그룹웨어 / 사내 시스템 연동
시나리오 3 · 그룹웨어

그룹웨어 / 사내 시스템 연동

이A

이준혁 (전략기획 과장) + 이수진 (IT 담당자)

결재 문서·주간보고가 그룹웨어에 쌓여 있음. 매주 보고서 작성 시 과거 자료 찾는 데 30분 이상 소요. IT 담당자의 API 연동 승인 필요.

2인 역할 분리 IT 승인 필요 보안 민감 도입 기간 1~2주
1
사용자

연동 요청 제출

'데이터 소스 추가 > 그룹웨어' 선택. 시스템 종류 선택 (더존 WEHAGO / 삼성SDS Brity / 카카오워크 / 기타). 연동 목적 입력 후 "IT 담당자에게 요청 보내기" 클릭.

사용자 관점 이 단계에서 할 일 끝. 이후는 IT 담당자 몫. "요청이 접수됐어요. 연결되면 알려드릴게요" 상태 메시지만 표시.
2
IT 담당자

관리자 포털에서 승인 + 허용 범위 설정

IT 담당자 이수진 씨는 관리자 포털에서 요청 목록 확인. 허용 데이터 범위 설정 (예: 결재 완료 문서만 / 2년 이내만). 승인 후 API 키 자동 생성 및 시스템에 등록.

설계 포인트 IT 담당자가 허용 범위를 세밀하게 제어할 수 있어야 함. "전체 연결"이 아니라 "이 사람은 이 범위만" 설정 가능.
3
사용자

연결 완료 알림 + 즉시 사용

"그룹웨어 연결이 완료됐어요" 알림 수신. 채팅에서 "지난달 전략기획 주간보고 요약해줘" 바로 질문 가능. 답변에 출처 링크 포함 (해당 결재 문서 직접 연결).

성공 지표 사용자가 직접 시스템 설정을 건드리지 않아도 사용 시작. 요청 → 승인 → 사용까지 사용자 개입 최소화.
4
사용자

접근 권한 실시간 상속

그룹웨어 원본 권한 그대로 반영. 이준혁 씨가 접근 불가한 문서는 AI도 참조 불가. "해당 문서는 접근 권한이 없어요"라고 자연어로 안내.

보안 설계 권한 체계를 OwlAI가 별도로 관리하지 않음. 원본 시스템 권한을 그대로 위임받는 구조가 보안 셀링포인트.
5
IT 담당자

실패 케이스 — API 변경 / 토큰 만료

연동이 끊기면 IT 담당자에게 먼저 알림. 사용자에게는 "그룹웨어 자료를 잠시 이용하기 어려워요. IT팀에서 확인 중이에요"로만 표시.

설계 포인트 에러 책임 소재를 명확히. 사용자 오류인지 시스템 문제인지 구분해서 안내하여 사용자에게 불필요한 부담을 주지 않는다.

세 시나리오 공통 설계 원칙

역할에 따라 화면 분리. 로컬 파일은 개인이 혼자 끝내고, NAS는 팀장 주도 + IT 보조, 그룹웨어는 사용자 요청만 하고 IT가 처리. 인터페이스가 역할을 자동 인식해 "지금 당신이 할 수 있는 것"만 보여준다.

실패 메시지에 다음 행동 포함. 에러가 나도 "이렇게 하면 돼요"가 나오면 사용자는 포기하지 않는다. 이탈이 가장 많이 발생하는 지점이 에러 화면.

원본 권한 상속이 핵심 차별화. OwlAI는 데이터가 외부 서버를 거치지 않기 때문에 원본 시스템의 권한 체계를 그대로 가져올 수 있다. 보안 민감 고객의 첫 번째 질문("내 권한 밖 자료는 AI도 못 보나요?")에 대한 직접 답변.
